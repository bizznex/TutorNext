# ContexureEarth Project Start Here

This repository currently contains the BizzNexx prototype and planning system that will become the first ContexureEarth product wedge.

## Current Product Direction

The product should move from:

```text
BizzNexx Learning Studio
```

to:

```text
ContexureEarth Tutor
```

The existing BizzNexx work should not be thrown away. It already contains the practical tutor operating loop:

- public trust page
- structured inquiry capture
- WhatsApp-first follow-up
- trial/demo workflow
- student/learner memory
- monthly fee tracker
- owner action queue
- proof/testimonials
- operator-assisted setup

The strategic shift is that ContexureEarth should go deeper into education, learner context, and learning intelligence instead of staying a generic service-business product.

## Active Execution Rule

Do not start broad feature work before resolving the production foundation:

- `C-001`: Audit current frontend architecture
- `D-001`: Audit current backend models/APIs
- `D-002`: Define tenant/account model
- `E-001`: Choose auth approach
- `N-001`: Choose production hosting architecture

These tasks protect the product from becoming a polished prototype that cannot safely handle real tutor/parent/student data.

## First Market Wedge

First ICP:

```text
Academic school tutor / small learning studio
```

First promise:

```text
Look credible. Capture better inquiries. Follow up on trials. Track fees politely. Grow through proof and referrals.
```

Long-term promise:

```text
Move from managing students to understanding learners.
```

## What To Preserve From BizzNexx

Preserve:

- workflow engine
- public page plus inquiry loop
- WhatsApp-first message handoff
- fee and follow-up actions
- setup/operator thinking
- beta review discipline
- deployment preparation

## What Must Change For ContexureEarth

Shift language and product center:

- Business -> Tutor Studio / Learning Studio
- Customer -> Learner
- Catalogue -> Courses / Batches
- Lead -> Parent Inquiry
- Payment -> Fee Cycle
- Growth Score -> Trust & Growth Readiness
- Health Score -> Operating Rhythm
- Beta Studio -> ContexureEarth Tutor

Add early:

- Learner Context Card
- Parent Progress Update
- education-specific privacy and consent language

## What Not To Build Yet

Do not lead with:

- full LMS
- parent/student app
- marketplace leads
- payment gateway
- heavy ERP
- GST/accounting
- corporate trainer workflows
- deep AI learner scoring

Those can be considered only after the tutor operating loop has paid traction.
